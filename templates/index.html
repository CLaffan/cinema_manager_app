Cinema App: Booking sub-system

This is a partial application created to present the booking subsystem for review. This sub-system was created by a different "developer" team as such differences between the main application and this sub-system will be evident.

Features:

The booking sub-system is able to
  • Provide a listing of bookings
  • Let the user create a booking
  • Modify and remove a booking
  • The booking data is saved to a database
The application is NOT a standalone application and requires integrating and dependencies.

Limitations & considerations:

Being a partial application, there are some missing features and design decisions applied;
  • Minimal styling and themeing applied - left up to the design team to deal with integration
  • Unit tests have not been included
  • No HTTPS support
  • No CSRF or XSS protection
  • A non multiuser relational database was used for the datastore.
  • Plain JavaScript and bootstrap
  • No session or cookies
  • Rudimentary sanitization and validation conducted on HTTP request & form variables
